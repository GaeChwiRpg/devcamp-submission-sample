# Evidence

이 폴더는 교재 `books/06-week10-backend-interview-survival.md`의 핵심 4 프레임(A-B-C 공식 + 꼬리질문 + "모를 때" + 기술 case study)이 본인 답변에 적용된 흔적을 남기는 곳입니다.

## 답변 자산

- `interview-script.md` — 미션 3개 이상 A-B-C 답변 (1분/3분 분량)
- `follow-up-questions.md` — 답변별 꼬리질문 + 본인 답안
- `presentation-notes.md` — 발표 슬라이드 흐름

## 책의 프레임 적용 자산

- `abc-answer-pack.md` — A-B-C 공식 적용 Before/After 비교 표
- `tech-qa-pack.md` — 인덱스/트랜잭션/캐시 기술 case study 답변 5개
- `dont-know-strategy.md` — "모를 때" 대처 답변 패턴 (책 3교시 2장 2절)

## 작성 원칙

- 모든 답변에 A-B-C 공식이 **명시적으로 표시**되어야 합니다 (`[A] ... [B] ... [C] ...`)
- 꼬리질문은 답변에서 자연스럽게 나올 만한 것만 — 본인이 답할 자신 없는 질문은 넣지 말 것
- "모를 때" 답변은 인정 + 본인 추론 + 학습 의지 3 단계
- 기술 답변은 "실무 case + 본인 경험" 결합 (책의 case study 형식 참고)
