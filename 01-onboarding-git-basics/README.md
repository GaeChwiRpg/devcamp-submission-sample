# 01-onboarding-git-basics

> 교재: `books/01-onboarding-git-basics.md` (1~9장 Git/GitHub 기초)

Week 1에 Spring Boot로 들어가기 전에 두 가지를 한 번씩 끝내 둡니다.

## 이번 주에 해야 하는 것

### Part A. Git/GitHub 제출 흐름 (책 1~9장)

- 제출 브랜치를 따로 파고 PR 여는 흐름을 손으로 한 번 직접 실행
- 커밋, push, PR 생성 순서를 명령 단위로 기록

### Part B. Java/IntelliJ/Gradle 환경 셀프체크

- JDK 21 설치 확인 (`java -version`)
- IntelliJ IDEA에서 Spring 프로젝트 import → Run 성공
- `./gradlew bootRun` 8080 포트 바인딩 확인

## 이번 주에 제출할 것

- `report.md`
- `evidence/git-flow-notes.md` — 브랜치/커밋/PR 흐름 정리 (Part A)
- `evidence/command-history.md` — 제출 과정 명령 예시 (Part A)
- `evidence/env-jdk-version.txt` — `java -version` 출력 (Part B)
- `evidence/intellij-run-and-bootrun-log.md` — IntelliJ Run + bootRun 로그 통합 (Part B)

## 평가 기준

- Git 명령 흐름이 명령 단위로 보이는가 (Part A)
- 환경 셀프체크 3가지가 실제로 통과했는가 (Part B)

## 미션 시작 전에 보면 좋은 것

1. 교재 `books/01-onboarding-git-basics.md` 1~9장
2. 이 README의 Part A·B 섹션
3. `evidence/README.md` — 어떤 자산을 남겨야 하는지
4. `report.md` — 본인 셀프체크 결과를 적는 템플릿
5. `docs/review-rubric.md`
