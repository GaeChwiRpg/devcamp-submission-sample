# 04-week3-backend-resume

> 교재: `books/04-week3-backend-resume-differentiation.md` (특히 1·4장 "탈락 vs 눈에 띄는 이력서" + "수치화 패턴")

이번 주는 단순히 이력서 문장을 바꾸는 게 아니라, **책의 "수치화 패턴" + "문제해결 서사"** 두 프레임을 본인 이력서에 적용하는 미션입니다.

## 책의 핵심 학습 포인트 (이번 주 적용 대상)

1. **탈락 이력서 vs 눈에 띄는 이력서** (책 1교시 1장): "사용했다/구현했다" → "왜 / 어떻게 / 결과 수치"
2. **면접관이 보는 경험 패턴** (책 1교시 2장): 문제해결능력을 보여주는 구조
3. **문제 상황 → 시도 → 결과** 서사 (책 1교시 3·4장): 매 bullet에 들어가야 할 3 요소
4. **수치화 패턴** (책 1교시 4장 11절): 막연한 "성능 향상" → "p95 198→41ms (4.8배)"

## 이번 주에 해야 하는 것

- 사전학습~Week 2 산출물을 이력서 bullet으로 변환
- before/after 비교: "사용했다" 류 → "문제 → 시도 → 결과 수치"
- **수치 출처를 본인 evidence 파일로 추적 가능하게** 적기 (Week 4 이후 추가 미션의 수치도 같은 방식)
- **문제해결 서사 1개 풀어쓰기**: "왜 이 문제가 일어났고 어떻게 풀었는지"를 면접 답변용 길이로
- 책의 "수치화 패턴" 표를 본인 이력서에 적용

## 평가 기준

- bullet 5개 이상 before/after 모두 작성
- 각 after bullet이 **문제 → 시도 → 결과 수치** 3 요소를 포함하는가
- 수치마다 출처 evidence 파일이 추적 가능한가
- 문제해결 서사 1개가 1분/3분 답변용으로 길이 가능한가
- 책의 "탈락 이력서" 패턴(사용했다/구현했다 류)이 after에서 사라졌는가
