# 01-onboarding-git-basics

> 이 미션은 강의 `AI한테 시키기 전에 알아야 할 Git 초기초`의 적용편입니다.

Week 1에 Spring Boot로 들어가기 전에 제출 브랜치 + PR 흐름을 한 번 손으로 끝내 둡니다.

## 이번 주에 해야 하는 것

- 제출 브랜치를 따로 파고 PR 여는 흐름을 손으로 한 번 직접 실행
- 커밋, push, PR 생성 순서를 명령 단위로 기록

## 이번 주에 제출할 것

- `report.md`
- `evidence/git-flow-notes.md` — 브랜치/커밋/PR 흐름 정리
- `evidence/command-history.md` — 제출 과정 명령 예시

## 평가 기준

- Git 명령 흐름이 명령 단위로 보이는가

## 미션 시작 전에 보면 좋은 것

1. 이 README의 "이번 주에 해야 하는 것" + "평가 기준"
2. `evidence/README.md` — 어떤 자산을 남겨야 하는지
3. `report.md` — 셀프체크 결과를 적는 템플릿
4. `docs/review-rubric.md`

(정답 PR은 미션 종료 후 운영진이 별도로 공유합니다.)

핵심은 "정답"이 아니라 **제출 흐름을 몸에 익히는 것**입니다. JDK/IntelliJ 환경 셀프체크는 02-week1-spring-boot 의 시작 전 단계로 옮겨졌습니다.
